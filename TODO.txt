	Это можно взять из проекта diskrete_tasks. Там мы используем сторонний модуль "check-disk-space"

2) проверить поведение ffmpeg, если ему дать файл другого типа, но сециально сделать ему расширение mp4, а также что будет если подсунуть просто другой файл, например архив.
	Виды ошибок:
	Когда читаем информацию о файле командой: ffmpeg -i video.avi
	Здесь мы можем отловить только 2 ошибки:
		1) Файла не существует:
			video.avi: No such file or directory
		
		2) Подсовываем НЕ ВИДЕО:
			schemas.pptx: Invalid data found when processing input
			
		3) Какому-нибудь архиву умышленно меняем расширение на mp4:
			[mov,mp4,m4a,3gp,3g2,mj2 @ 0000000000450980] Format mov,mp4,m4a,3gp,3g2,mj2 detected only with low score of 1, misdetection possible!
			[mov,mp4,m4a,3gp,3g2,mj2 @ 0000000000450980] moov atom not found
			out2.mp4: Invalid data found when processing input
		
		4) Битый файл mp4. Т.е. открыли его в блокноте и удалили кусок текста из центра:
			[mov,mp4,m4a,3gp,3g2,mj2 @ 0000000000430980] moov atom not found
			out2.mp4: Invalid data found when processing input
	Т.е. в принципе 2 ошибки:
		а) No such file or directory
		б) Invalid data found when processing input

3) проблемка насчет request_id от пользователя: если он не уникальный, то мы можем удалить лишнего. Решение 1: в lb_d_task_done() помимо сверки request_id, сравнивать также status == 'work'
